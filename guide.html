Copilot Usage Tracker
🔑 GitHub 토큰 발급 가이드
Fine-grained Personal Access Token 기준 · Plan: Read-only 권한 필요
1
Fine-grained Token 페이지 열기
GitHub에 로그인한 상태에서 우측 상단 프로필 → Settings → Developer settings → Personal access tokens → Fine-grained tokens 순서로 이동합니다.
↗ Fine-grained tokens 바로가기
⚠ Classic token은 이 API를 지원하지 않습니다. 반드시 Fine-grained token을 사용하세요.
2
새 토큰 생성
"Generate new token" 버튼을 클릭합니다.
# Token name 예시
Copilot Usage Tracker
3
Resource owner 및 만료 기간 설정
Resource owner는 본인 계정을 선택합니다. Expiration은 90 days를 권장합니다.
✓ 본인 계정 (개인) Resource owner로 선택
✓ 90 days 권장 만료 기간
4
권한(Permissions) 설정
Account permissions 섹션에서 아래 항목을 찾아 설정합니다.
✓ Plan Account permissions → Plan → Read-only 필수
✓ 그 외 모든 권한 이 도구에서는 불필요 설정 불필요
⚠ Repository access는 "All repositories"가 아닌 "Public Repositories (read-only)"로 두어도 충분합니다.
5
토큰 생성 및 복사
페이지 하단 "Generate token" 클릭 후 즉시 복사하세요. 이 화면에서만 확인 가능합니다.
# Fine-grained token 형식
github_pat_xxxxxxxxxxxxxxxxxxxxxxxxxxxx
⚠ Classic token(ghp_로 시작)과 다릅니다. github_pat_으로 시작하는 토큰인지 확인하세요.
6
.env 파일에 입력
복사한 토큰을 .env 파일의 GITHUB_TOKEN에 입력하고 서버를 재시작하면 됩니다.
GITHUB_TOKEN=github_pat_복사한토큰
GITHUB_USERNAME=본인유저네임
💡 Enhanced Billing이 활성화된 유료 플랜(Pro/Business/Enterprise) 계정에서만 조회 가능합니다. github.com/settings/billing 좌측에 "Premium request analytics" 메뉴가 보이면 활성화된 상태입니다.
🔒 토큰은 .env 파일에만 보관되며 외부로 전송되지 않습니다
